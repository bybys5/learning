$scope对象的职责是承载DOM中指令多共享的操作和模型

  • 操作指的是$scope上的标准JavaScript方法

  • 模型指的是$scope上保存的包含瞬时状态数据的JavaScript对象。 持久化状态的数据应该保存在服务中，服务的作用是处理模型的持久化

  • 出于技术和架构方面的原因，绝对不要直接将控制器中$scope赋值为值类型对象（字符串、布尔值或数字）。 DOM中应该始终通过点操作符来访问数据。

  • 控制器应该尽可能简单。虽然可以用控制器来组织所有的功能，但是将业务逻辑移到服务和指令中是非常好的注意
